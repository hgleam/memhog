# memhog — 依頼者向け仕様書

最終更新: 2026-07-10

## 概要

macOS の**実メモリ（物理フットプリント）を最も食っているプロセスを特定して提示する**診断 CLI。

`ps aux` の RSS は Metal/MPS（Apple Silicon の GPU 共有＝ユニファイドメモリ）を数えないため、
ComfyUI 等の ML 系 Python は RSS 上「数十 MB」に見えるのに実際は数十 GB を常駐している。
memhog は Activity モニタ相当の物理フットプリント（`top` の MEM 列）でランク付けし、RSS との
乖離が大きいプロセスに `⚠ GPU/Metal常駐(psに出ない)` 印を付けて「小さく見えるのに実は巨大」を炙り出す。

## 主要機能

| 機能 | 説明 | 状態 |
|------|------|------|
| 実メモリ上位表示 | 物理フットプリント降順でプロセスを一覧（既定 15 件） | ✅ 実装済み |
| GPU/Metal 常駐の検出 | RSS に現れない GPU 共有メモリを抱えるプロセスに警告印 | ✅ 実装済み |
| コマンドフィルタ | フルコマンドへの部分一致（大小無視）で絞り込み（`-g`） | ✅ 実装済み |
| JSON 出力 | 機械可読な JSON（通知連携・定期実行向け）（`--json`） | ✅ 実装済み |
| 監視モード | 指定秒間隔で画面を更新し続ける（`--watch`） | ✅ 実装済み |
| プロセス停止 | 一覧から PID を選んで停止（既定で確認、不可逆）（`--kill`） | ✅ 実装済み |
| システムメモリ表示 | PhysMem・Swap・空き割合を併記 | ✅ 実装済み |

## 利用方法

```bash
memhog                 # 実メモリ上位15件
memhog -n 30           # 上位30件
memhog -g python       # フルコマンドに "python" を含むものだけ
memhog --json          # 機械可読 JSON
memhog --watch 2       # 2秒ごとに更新し続ける監視モード（Ctrl-C で終了）
memhog --kill          # 一覧から PID を選んで停止（既定で確認、不可逆操作）
memhog --kill --force  # SIGKILL で停止
memhog --kill -y       # 確認を省略して停止
memhog --version       # バージョン表示
```

セットアップ手順は [開発者向け仕様書](../develop/README.md) を参照。

## 制約・受け入れ基準

- **macOS 専用**。`top` / `ps` / `sysctl` / `memory_pressure` に依存する（Linux では動作しない）。
- 外部コマンドが失敗しても例外は投げず、取得できた範囲で表示する（空欄になる）。
- `--kill` は不可逆操作。既定で確認プロンプトを出し、PID ≤ 1・自分自身は停止不可。
- `--watch` は `--json` / `--kill` と併用できない。
- 権限不足（他ユーザーのプロセス等）の停止は失敗し、その旨を表示する。
